프로젝트 소개

11개 병원 응급상황실을 실시간 양방향 대시보드로 연결하여, 전화 기반 전원 의뢰의 초기 대응 지연을 해소하는 1단계 웹 플랫폼(MVP)을 구축합니다.

프로젝트 개요

항목내용
프로젝트명대전권역 병원 간 응급환자 전원 연계망 웹 시스템 구축

핵심 기술React + Next.js(SPA) · NestJS · Socket.IO(WebSocket) · PostgreSQL · AWS · FCM

핵심 요구사항 분석

  • WebSocket(Socket.IO) 기반 실시간 양방향 전원 의뢰·수용 대시보드 — 브라우저 새로고침 없는 즉시 갱신
  • 11개 병원 상황실 상시 접속 환경 대응 — 소켓 끊김 방지 및 자동 재연결 로직
  • 공공 API(E-Gen, 중앙응급의료센터 응급상황실) 연동 + 병원별 자원(병상·CT·MRI·당직) 수기 입력 관리
  • 5개 역할(의사·간호사·구급대원·병원 관리자·총괄 관리자) 차등 권한 및 메뉴 접근 분리
  • 의료기관 소속 인증·가입 승인 절차 + 환자 개인정보 보안(암호화 저장·접근 감사)
  • 응급실 환경에 맞춘 고가시성 반응형 SPA UI/UX
  • 2단계 고도화(모바일 앱·전국 챗봇) 대비 FE/BE 완전 분리 RESTful 아키텍처

제안 포인트

1. 실시간 양방향 동기화 — 검증된 실시간 처리 경험. 투자조합 관리 플랫폼과 전자결재 SaaS에서 CRDT(Yjs) 기반 실시간 공동 편집을 구현하며 다수 사용자의 동시 입력을 지연 없이 동기화한 경험이 있습니다. 본 프로젝트에서는 Socket.IO 기반 양방향 채널로 전원 요청·확인·수용·거절·보류 상태를 즉시 전파하고, 상시 접속 환경을 가정한 하트비트·자동 재연결·메시지 재전송 큐를 적용합니다.

2. 공공 의료 API 연동. 다수의 외부 API·규제 시스템 연동(NICE 본인인증, VICS 규제 보고 SFTP 자동 제출 등) 경험을 바탕으로, E-Gen(중앙응급의료센터) 공개 데이터를 안정적으로 수집·캐싱하여 병원별 일반 현황과 수기 입력 자원 현황을 단일 화면에 통합합니다.

3. 의료급 보안과 권한 관리. 6계층 보안 + 7종 RBAC(전자결재 SaaS), 5중 보안 + NICE 본인인증(투자조합 플랫폼) 등 민감정보 보호 경험을 적용하여, 5개 역할별 메뉴 접근 분리, 의료기관 소속 인증, 환자 정보 암호화·접근 감사 로그를 설계합니다.

4. 확장 대비 아키텍처. FE/BE를 완전 분리한 RESTful + WebSocket 구조로, 2단계 모바일 앱·전국 단위 챗봇 확장 시 소스 전면 재작성 없이 인터페이스 재사용이 가능하도록 설계합니다.

PRD (Product Requirements Document)

프로젝트 요구사항을 정의하고 기능 범위를 명확히 합니다.

1. 프로젝트 배경 및 목적

현재 대전권역 11개 병원 간 응급환자 전원·수용 과정이 유선 전화로 개별 진행되어 응급 상황 초기 대응이 지연되고 있습니다. 본 프로젝트는 각 병원 응급상황실의 환자 상태, 병상 가용성, 의료 자원 데이터를 통합하여 웹 브라우저에서 즉각적인 전원 의뢰와 수용 여부를 결정할 수 있는 실시간 플랫폼(1단계 MVP)을 구축합니다. 금년 11월 시범 운영을 시작하여 늦어도 12월 시스템 가동을 목표로 합니다.

3. 기능 요구사항

3.1 회원가입 및 권한 관리

  • [필수] 병원 소속 담당자 가입 신청(이메일·소속 정보) 및 총괄 관리자 승인
  • [필수] 5개 그룹 차등 권한 부여 및 메뉴 접근 분리(RBAC)
  • [필수] 의료기관 소속 확인 인증 및 로그인 세션 보안 유지

3.2 응급의료 자원 현황 조회

  • [필수] E-Gen(중앙응급의료센터) 공개 데이터 연동 — 병원별 일반 현황 출력
  • [필수] 병원별 가용 병상·CT·MRI·당직 전문의 현황 수기 입력·수정
  • [필수] 전체 병원 수용 가능 상태 통합 조회 리스트

3.3 실시간 전원 의뢰 및 양방향 대시보드

  • [필수] 환자 상태·의뢰 사유 기입 전원 요청서 작성·발송
  • [필수] 송신·수신·수용·거절·보류 상태 대시보드 실시간 출력
  • [필수] WebSocket 기반 새로고침 없는 양방향 데이터 동기화
  • [필수] 전원 의뢰 발생 시 수신 병원 실시간 푸시 알림(FCM)

5. 기술 제약사항

  • 프론트엔드: 실시간 렌더링을 위한 SPA 프레임워크 기반(React + Next.js)
  • 백엔드: 비동기 실시간 처리에 적합한 Node.js 기반(NestJS) + Socket.IO
  • 인프라: 확장이 용이한 클라우드(AWS) — 2단계 병원 수 확장 대비
  • 외부 의존: E-Gen 공공 API 사양·쿼터에 따른 폴링·캐싱 전략 적용

6. 범위 외 사항 (Out of Scope)

  • 2단계 고도화: 모바일 애플리케이션 연동
  • 2단계 고도화: 전국 단위 챗봇 기능 확장
  • 병원 내부 EMR/HIS 직접 연동(1단계는 수기 입력 + 공공 API 기준)
