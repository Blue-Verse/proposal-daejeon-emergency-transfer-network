### 6-4. 클라이언트 질문 답변 (복사용)

**Q1. 실시간 데이터 처리가 가능한 프로젝트 경험이 있나요?**

네, 다수의 실시간 데이터 처리 시스템을 구축해 왔습니다. 투자조합 관리 플랫폼과 전자결재 SaaS에서 CRDT(Yjs) 기반 실시간 공동 편집을 구현하여 여러 사용자의 동시 입력을 지연 없이 동기화한 경험이 있으며, 시니어 주간보호 관리 플랫폼에서는 돌봄 기록과 알림을 실시간으로 전달하는 구조를 설계했습니다. 본 프로젝트의 핵심인 WebSocket(Socket.IO) 기반 양방향 동기화는 브라우저 새로고침 없이 전원 요청·수용 상태를 즉시 갱신하며, 11개 병원 상황실의 상시 접속 환경을 가정해 소켓 끊김 방지와 자동 재연결 로직까지 구현하겠습니다. (약 300자)

**Q2. 실시간 채팅 및 push 알림 기능이 포함된 대시보드 형태의 시스템 구축 경험이 있나요?**

네, 보유하고 있습니다. 시니어 주간보호 관리 플랫폼에서 Firebase Cloud Messaging(FCM) 기반 실시간 푸시 알림과 보호자 소통 기능을 구현하여 돌봄 상황을 즉시 전달했고, 전자결재 SaaS와 투자조합 플랫폼에서는 결재·승인 상태를 실시간으로 갱신하는 관리자 대시보드를 설계·운영했습니다. 본 프로젝트에서는 전원 의뢰가 발생하면 수신 병원에 즉시 푸시 알림을 전송하고, 송신·수신·수용·거절·보류 상태를 단일 대시보드 목록에 실시간 반영하며, 병원 간 의뢰 사유 전달을 위한 메시지 기능을 함께 제공하겠습니다. (약 290자)

**Q3. 개인정보 관련 강력한 보안과 계정별 접근 권한 관리가 포함된 프로젝트 경험이 있나요?**

네, 금융·의료 등 민감정보를 다루는 시스템에서 강력한 보안 체계를 구축해 왔습니다. 전자결재 SaaS는 JWT·2FA·IP 화이트리스트·Rate Limiting을 포함한 6계층 보안과 7종 역할별 권한(RBAC)을 적용했고, 투자조합 플랫폼은 NICE 본인인증·GeoIP·DLP를 포함한 5중 보안으로 금융 데이터를 보호했습니다. 본 프로젝트에서는 의사·간호사·구급대원·병원 관리자·총괄 관리자 5개 역할별 메뉴 접근 권한을 분리하고, 의료기관 소속 인증과 가입 승인 절차, 환자 정보 암호화 저장 및 접근 감사 로그를 적용하여 개인정보를 안전하게 관리하겠습니다. (약 320자)
